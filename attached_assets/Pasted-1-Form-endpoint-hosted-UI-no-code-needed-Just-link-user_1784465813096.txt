https://talaosman771.app.n8n.cloud/form/financial-statement-analyser

2. API endpoint (for your own frontend)
This is the one you'll wire up in code.

POST https://talaosman771.app.n8n.cloud/webhook/financial-analysis
Content-Type: application/json
Note: while the workflow is unpublished/in test mode, the URL is /webhook-test/financial-analysis and only fires once per "Listen for test event" click. Once you publish the workflow, use the production /webhook/... path above.

Request body
The Prepare Financials (API) node accepts flexible shapes. Send:

{
  "periodLabel": "FY2024",
  "notes": "Optional context: one-offs, assumptions, business events",
  "financialData": [
    { "label": "Revenue", "value": 1250000 },
    { "label": "Gross Profit", "value": 470000 },
    { "label": "Net Income", "value": 95000 },
    { "label": "Total Current Assets", "value": 340000 },
    { "label": "Inventory", "value": 90000 },
    { "label": "Total Current Liabilities", "value": 200000 },
    { "label": "Total Liabilities", "value": 520000 },
    { "label": "Total Shareholders' Equity", "value": 480000 },
    { "label": "Cash from Operating Activities", "value": 130000 }
  ]
}
financialData also accepts: a plain string, an array of arrays ([["Revenue", 1250000], ...]), or a nested object. Keys financials, data, or statements work as aliases; periodLabel also accepts period / Period Label.

Example frontend call
async function analyze(payload) {
  const res = await fetch(
    "https://talaosman771.app.n8n.cloud/webhook/financial-analysis",
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(payload),
    }
  );
  const { report, dashboard_url } = await res.json();
  return { report, dashboard_url };
  // report.risks (3), report.opportunities (3), report.recommendations (4 areas)
}
